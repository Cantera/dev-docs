46 //! Import a reaction mechanism for a phase or an interface.
47 /*!
48  * This routine will import a reaction mechanism into a kinetics object. The
49  * reaction mechanism may either be homogeneous or heterogeneous, involving
50  * multiple ThermoPhase objects. The hosting phase should be included as the
51  * first argument. For example, if phase I is an interface phase between bulk
52  * phases A and B. Then, the XML_Node for phase I should be the first
53  * argument. The vector of ThermoPhase objects should consist of pointers to
54  * phases I, A, and B.
55  *
56  * @param phase This is an XML node containing a description of the owning
57  * phase for the kinetics object. Within the phase is a XML
58  * element called reactionArray containing the location of the
59  * description of the reactions that make up the kinetics object.
60  * Also within the phase is an XML element called phaseArray
61  * containing a listing of other phases that participate in the
62  * kinetics mechanism.
63  *
64  * @param th This is a list of ThermoPhase pointers which must include all
65  * of the phases that participate in the kinetics operator. All
66  * of the phases must have already been initialized and formed
67  * within Cantera. However, their pointers should not have been
68  * added to the Kinetics object; this addition is carried out
69  * here. Additional phases may be include in the list; these have
70  * no effect.
71  *
72  * @param kin This is a pointer to a kinetics manager class that will be
73  * initialized with the kinetics mechanism. Inherited Kinetics
74  * classes should be used here.
75  *
76  * @ingroup kineticsmgr
77  */
78 bool importKinetics(const XML_Node& phase, std::vector<ThermoPhase*> th,
79  Kinetics* kin);
