158 //! Import a reaction mechanism for a phase or an interface.
159 /*!
160  * This routine will import a reaction mechanism into a kinetics object. The
161  * reaction mechanism may either be homogeneous or heterogeneous, involving
162  * multiple ThermoPhase objects. The hosting phase should be included as the
163  * first argument. For example, if phase I is an interface phase between bulk
164  * phases A and B. Then, the XML_Node for phase I should be the first
165  * argument. The vector of ThermoPhase objects should consist of pointers to
166  * phases I, A, and B.
167  *
168  * @param phase This is an xml node containing a description of the owning
169  * phase for the kinetics object. Within the phase is a XML
170  * element called reactionArray containing the location of the
171  * description of the reactions that make up the kinetics object.
172  * Also within the phase is an XML element called phaseArray
173  * containing a listing of other phases that participate in the
174  * kinetics mechanism.
175  *
176  * @param th This is a list of ThermoPhase pointers which must include all
177  * of the phases that participate in the kinetics operator. All
178  * of the phases must have already been initialized and formed
179  * within Cantera. However, their pointers should not have been
180  * added to the Kinetics object; this addition is carried out
181  * here. Additional phases may be include in the list; these have
182  * no effect.
183  *
184  * @param kin This is a pointer to a kinetics manager class that will be
185  * initialized with the kinetics mechanism. Inherited Kinetics
186  * classes should be used here.
187  *
188  * @ingroup kineticsmgr
189  *
190  */
