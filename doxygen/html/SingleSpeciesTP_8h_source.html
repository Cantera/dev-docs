20 /**
21  * @ingroup thermoprops
22  *
23  * The SingleSpeciesTP class is a filter class for ThermoPhase. What it does is
24  * to simplify the construction of ThermoPhase objects by assuming that the
25  * phase consists of one and only one type of species. In other words, it's a
26  * stoichiometric phase. However, no assumptions are made concerning the
27  * thermodynamic functions or the equation of state of the phase. Therefore it's
28  * an incomplete description of the thermodynamics. The complete description
29  * must be made in a derived class of SingleSpeciesTP.
30  *
31  * Several different groups of thermodynamic functions are resolved at this
32  * level by this class. For example, All partial molar property routines call
33  * their single species standard state equivalents. All molar solution
34  * thermodynamic routines call the single species standard state equivalents.
35  * Activities routines are resolved at this level, as there is only one species.
36  *
37  * It is assumed that the reference state thermodynamics may be obtained by a
38  * pointer to a populated species thermodynamic property manager class (see
39  * ThermoPhase::m_spthermo). How to relate pressure changes to the reference
40  * state thermodynamics is again left open to implementation.
41  *
42  * Mole fraction and Mass fraction vectors are assumed to be equal to x[0] = 1
43  * y[0] = 1, respectively. Simplifications to the interface of setState_TPY()
44  * and setState_TPX() functions result and are made within the class.
45  *
46  * Note, this class can handle the thermodynamic description of one phase of one
47  * species. It can not handle the description of phase equilibrium between two
48  * phases of a stoichiometric compound (e.g. water liquid and water vapor, below
49  * the critical point). However, it may be used to describe the thermodynamics
50  * of one phase of such a compound even past the phase equilibrium point, up to
51  * the point where the phase itself ceases to be a stable phase.
52  *
53  * This class doesn't do much at the initialization level. Its
54  * SingleSpeciesTP::initThermo() member does check that one and only one species
55  * has been defined to occupy the phase.
56  */
