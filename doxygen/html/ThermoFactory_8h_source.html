163 //! Import a phase information into an empty ThermoPhase object
164 /*!
165  * Here we read an XML description of the thermodynamic information
166  * for a phase. At the end of this routine, the phase should
167  * be ready to be used within applications. This routine contains
168  * some key routines that are used as pass back routines so that
169  * the phase (and the contents of the XML file) may contain
170  * variable parameterizations for the specification of the
171  * species standard states, the equation of state, and the
172  * specification of other nonidealities. Below, a description
173  * is presented of the main algorithm for bringing up a ThermoPhase
174  * object, with care to present points where customizations
175  * occur.
176  *
177  * Before invoking this routine, either the ThermoPhase Factory routines
178  * are called or direct constructor routines are called that
179  * instantiate an inherited ThermoPhase object. This object is input
180  * to this routine, and therefore contains inherited routines that
181  * drive the customization of the initialization process.
182  *
183  * At the start of the routine, we import descriptions of the elements
184  * that make up the species in a phase.
185  *
186  * We call setParametersFromXML(eos) to read parameters about
187  * the thermo phase before the species are read in.
188  *
189  * We call addElementsFromXML() to add elements into the
190  * description of the phase.
191  *
192  * We create a new species thermo manager. Function
193  * 'newSpeciesThermoMgr' looks at the species in the database
194  * to see what thermodynamic property parameterizations are
195  * used, and selects a class that can handle the
196  * parameterizations found.
197  *
198  * We import information about the species, including their
199  * reference state thermodynamic polynomials. We then freeze
200  * the state of the species in the element.
201  *
202  * Finally, we call initThermoXML(),
203  * a member function of the ThermoPhase object, to "finish"
204  * the description. Now that the species are known,
205  * additional information may be read in about the thermodynamics
206  * of the phase, (e.g., virial coefficients, which are
207  * binary or ternary interaction parameters between species).
208  *
209  * @param phase This object must be the phase node of a
210  * complete XML tree
211  * description of the phase, including all of the
212  * species data. In other words while "phase" must
213  * point to an XML phase object, it must have
214  * sibling nodes "speciesData" that describe
215  * the species in the phase.
216  * @param th Pointer to the ThermoPhase object which will
217  * handle the thermodynamics for this phase.
218  * We initialize part of the ThermoPhase object
219  * here, especially for those objects which are
220  * part of the Cantera Kernel.
221  *
222  * @param spfactory species Thermo factory pointer, if
223  * available. If not available, one will be
224  * created.
225  * @ingroup thermoprops
226  *
227  */
228 void importPhase(XML_Node& phase, ThermoPhase* th);
