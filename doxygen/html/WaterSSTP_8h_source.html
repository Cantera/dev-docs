23 //! Class for single-component water. This is designed to cover just the liquid
24 //! part of water.
25 /*!
26  * The reference is W. Wagner, A. Prub, "The IAPWS Formulation 1995 for the
27  * Thermodynamic Properties of Ordinary Water Substance for General and
28  * Scientific Use," J. Phys. Chem. Ref. Dat, 31, 387, 2002.
29  *
30  * <HR>
31  * <H2> Specification of Species Standard State Properties </H2>
32  * <HR>
33  *
34  * The offsets used in the steam tables are different than NIST's. They assume
35  * u_liq(TP) = 0.0, s_liq(TP) = 0.0, where TP is the triple point conditions:
36  *
37  * - u(273.16, rho) = 0.0
38  * - s(273.16, rho) = 0.0
39  * - psat(273.16) = 611.655 Pascal
40  * - rho(273.16, psat) = 999.793 kg m-3
41  *
42  * These "steam table" assumptions are used by the WaterPropsIAPWS class.
43  * Therefore, offsets must be calculated to make the thermodynamic properties
44  * calculated within this class to be consistent with thermo properties within
45  * Cantera.
46  *
47  * The thermodynamic base state for water is set to the NIST basis here by
48  * specifying constants, #EW_Offset and #SW_Offset, one for energy quantities
49  * and one for entropy quantities. The offsets are specified so that the
50  * following properties hold:
51  *
52  * - Delta_Hfo_idealgas(298.15) = -241.826 kJ/gmol
53  * - So_idealgas(298.15, 1bar) = 188.835 J/gmolK
54  *
55  * (From http://webbook.nist.gov)
56  *
57  * The "o" here refers to a hypothetical ideal gas state. The way we achieve
58  * this in practice is to evaluate at a very low pressure and then use the
59  * theoretical ideal gas results to scale up to higher pressures:
60  *
61  * Ho(1bar) = H(P0)
62  *
63  * So(1bar) = S(P0) + RT ln(1bar/P0)
64  *
65  * <HR>
66  * <H2> %Application within Kinetics Managers </H2>
67  * <HR>
68  *
69  * This is unimplemented.
70  *
71  * <HR>
72  * <H2> Instantiation of the Class </H2>
73  * <HR>
74  *
75  * The constructor for this phase is NOT located in the default ThermoFactory
76  * for %Cantera. However, a new WaterSSTP object may be created by the following
77  * code snippets, combined with an XML file given in the XML example section.
78  *
79  * @code
80  * WaterSSTP *w = new WaterSSTP("waterSSTPphase.xml","");
81  * @endcode
82  *
83  * or
84  *
85  * @code
86  * XML_Node *xm = get_XML_NameID("phase", "waterSSTPphase.xml#water", 0);
87  * WaterSSTP *w = new WaterSSTP(*xm);
88  * @endcode
89  *
90  * or by the following call to importPhase():
91  *
92  * @code
93  * XML_Node *xm = get_XML_NameID("phase", "waterSSTPphase.xml#water", 0);
94  * WaterSSTP water;
95  * importPhase(*xm, &water);
96  * @endcode
97  *
98  * <HR>
99  * <H2> XML Example </H2>
100  * <HR>
101  *
102  * An example of an XML Element named phase setting up a WaterSSTP object with
103  * id "water" is given below.
104  *
105  * @code
106  * <!-- phase water -->
107  * <phase dim="3" id="water">
108  * <elementArray datasrc="elements.xml">O H </elementArray>
109  * <speciesArray datasrc="#species_data">H2O</speciesArray>
110  * <state>
111  * <temperature units="K">300.0</temperature>
112  * <pressure units="Pa">101325.0</pressure>
113  * </state>
114  * <thermo model="PureLiquidWater"/>
115  * <kinetics model="none"/>
116  * </phase>
117  * @endcode
118  *
119  * Note the model "PureLiquidWater" indicates the usage of the WaterSSTP object.
120  *
121  * @ingroup thermoprops
122  */
