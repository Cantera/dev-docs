104 //! Base class for transport property managers.
105 /*!
106  * All classes that compute transport properties for a single phase
107  * derive from this class. Class
108  * %Transport is meant to be used as a base class only. It is
109  * possible to instantiate it, but its methods throw exceptions if
110  * called.
111  *
112  * Note, transport properties for multiphase situations have yet to be
113  * fully developed within Cantera.
114  *
115  * All member functions are virtual, unless otherwise stated.
116  *
117  * <HR>
118  * <H2> Relationship of the %Transport class to the %ThermoPhase Class </H2>
119  * <HR>
120  *
121  * This section describes how calculations are carried out within
122  * the %Transport class. The %Transport class and derived classes of the
123  * the %Transport class necessarily use the %ThermoPhase class to obtain
124  * the list of species and the thermodynamic state of the phase.
125  *
126  * No state information is stored within %Transport classes.
127  * Queries to the underlying ThermoPhase object must be made to obtain
128  * the state of the system.
129  *
130  * An exception to this however is the state information concerning the
131  * the gradients of variables. This information is not stored within
132  * the ThermoPhase objects. It may be collected within the Transport objects.
133  * In fact, the meaning of const operations within the Transport class
134  * refers to calculations which do not change the state of the
135  * system nor the state of the first order gradients of the system.
136  *
137  * When a const operation is evoked within the Transport class, it is
138  * also implicitly assumed that the underlying state within the ThermoPhase
139  * object has not changed its values.
140  *
141  *
142  * <HR>
143  * <H2> Diffusion Fluxes and their Relationship to Reference Velocities </H2>
144  * <HR>
145  *
146  * The diffusion fluxes must be referenced to a particular reference
147  * fluid velocity.
148  * Most typical is to reference the diffusion fluxes to the mass averaged velocity, but
149  * referencing to the mole averaged velocity is suitable for some
150  * liquid flows, and referencing to a single species is suitable for
151  * solid phase transport within a lattice. Currently, the identity of the reference
152  * velocity is coded into each transport object as a typedef named VelocityBasis, which
153  * is equated to an integer. Negative values of this variable refer to mass or mole-averaged
154  * velocities. Zero or positive quantities refers to the reference
155  * velocity being referenced to a particular species. Below are the predefined constants
156  * for its value.
157  *
158  * - VB_MASSAVG Diffusion velocities are based on the mass averaged velocity
159  * - VB_MOLEAVG Diffusion velocities are based on the mole averaged velocities
160  * - VB_SPECIES_0 Diffusion velocities are based on the relative motion wrt species 0
161  * - ...
162  * - VB_SPECIES_3 Diffusion velocities are based on the relative motion wrt species 3
163  *
164  * All transport managers specify a default reference velocity in their default constructors.
165  * All gas phase transport managers by default specify the mass-averaged velocity as their
166  * reference velocities.
167  *
168  *
169  * @todo Provide a general mechanism to store the gradients of state variables
170  * within the system.
171  *
172  * @ingroup tranprops
173  */

356  //! Returns the self diffusion coefficients of the species in the phase
357  /*!
358  * The self diffusion coefficient is the diffusion coefficient of a tracer species
359  * at the current temperature and composition of the species. Therefore,
360  * the dilute limit of transport is assumed for the tracer species.
361  * The effective formula may be calculated from the stefan-maxwell formulation by
362  * adding another row for the tracer species, assigning all D's to be equal
363  * to the respective species D's, and then taking the limit as the
364  * tracer species mole fraction goes to zero. The corresponding flux equation
365  * for the tracer species k in units of kmol m-2 s-1 is.
366  *
367  * \f[
368  * J_k = - D^{sd}_k \frac{C_k}{R T} \nabla \mu_k
369  * \f]
370  *

372  *
373  * The self diffusion calculation is handled by subclasses of
374  * LiquidTranInteraction as specified in the input file.
375  * These in turn employ subclasses of LTPspecies to
376  * determine the individual species self diffusion coeffs.
377  *
378  * @param selfDiff Vector of self-diffusion coefficients
