333  //! Bound the Newton step while relaxing the solution
334  /*!
335  * Return the factor by which the undamped Newton step 'step0'
336  * must be multiplied in order to keep all solution components in
337  * all domains between their specified lower and upper bounds.
338  * Other bounds may be applied here as well.
339  *
340  * Currently the bounds are hard coded into this routine:
341  *
342  * Minimum value for all variables: - 0.01 * m_ewt[i]
343  * Maximum value = none.
344  *
345  * Thus, this means that all solution components are expected
346  * to be numerical greater than zero in the limit of time step
347  * truncation errors going to zero.
348  *
349  * Delta bounds: The idea behind these is that the Jacobian
350  * couldn't possibly be representative if the
351  * variable is changed by a lot. (true for
352  * nonlinear systems, false for linear systems)
353  * Maximum increase in variable in any one newton iteration: factor of 2
354  * Maximum decrease in variable in any one newton iteration: factor of 5
355  *
356  * @param y Current value of the solution
357  * @param step0 Current raw step change in y[]
358  * @param loglevel Log level. This routine produces output if loglevel
359  * is greater than one
360  *
361  * @return Returns the damping coefficient
362  */
363  double boundStep(const double* const y, const double* const step0, int loglevel);
