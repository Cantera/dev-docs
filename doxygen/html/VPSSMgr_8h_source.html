27 /**
28  * @defgroup mgrpdssthermocalc Managers for Calculating Standard-State
29  * Thermodynamics
30  *
31  * To compute the thermodynamic properties of multicomponent solutions, it is
32  * necessary to know something about the thermodynamic properties of the
33  * individual species present in the solution. Exactly what sort of species
34  * properties are required depends on the thermodynamic model for the solution.
35  * For a gaseous solution (i.e., a gas mixture), the species properties required
36  * are usually ideal gas properties at the mixture temperature and at a
37  * reference pressure (almost always at 1 bar). For other types of solutions,
38  * however, it may not be possible to isolate the species in a "pure" state. For
39  * example, the thermodynamic properties of, say, Na+ and Cl- in saltwater are
40  * not easily determined from data on the properties of solid NaCl, or solid Na
41  * metal, or chlorine gas. In this case, the solvation in water is fundamental
42  * to the identity of the species, and some other reference state must be used.
43  * One common convention for liquid solutions is to use thermodynamic data for
44  * the solutes in the limit of infinite dilution within the pure solvent;
45  * another convention is to reference all properties to unit molality.
46  *
47  * In defining these standard states for species in a phase, we make the
48  * following definition. A reference state is a standard state of a species in a
49  * phase limited to one particular pressure, the reference pressure. The
50  * reference state specifies the dependence of all thermodynamic functions as a
51  * function of the temperature, in between a minimum temperature and a maximum
52  * temperature. The reference state also specifies the molar volume of the
53  * species as a function of temperature. The molar volume is a thermodynamic
54  * function. A full standard state does the same thing as a reference state, but
55  * specifies the thermodynamics functions at all pressures.
56  *
57  * Class VPSSMgr is the base class for a family of classes that compute
58  * properties of all species in a phase in their standard states, for a range
59  * of temperatures and pressures.
60  *
61  * Phases which use the VPSSMGr class must have their respective ThermoPhase
62  * objects actually be derivatives of the VPStandardState class. These classes
63  * assume that there exists a standard state for each species in the phase,
64  * where the Thermodynamic functions are specified as a function of
65  * temperature and pressure. Standard state thermo objects for each species
66  * in the phase are all derived from the PDSS virtual base class. Calculators
67  * for these standard state thermo , which coordinate the calculation for all
68  * of the species in a phase, are all derived from VPSSMgr. In turn, these
69  * standard states may employ reference state calculation to aid in their
70  * calculations. And the VPSSMgr calculators may also employ SimpleThermo
71  * calculators to help in calculating the properties for all of the species in
72  * a phase. However, there are some PDSS objects which do not employ reference
73  * state calculations. An example of this is a real equation of state for
74  * liquid water used within the calculation of brine thermodynamics.
75  *
76  * Typically calls to calculate standard state thermo properties are virtual
77  * calls at the ThermoPhase level. It is left to the child classes of
78  * ThermoPhase to specify how these are carried out. Usually, this will involve
79  * calling the m_spthermo pointer to a SpeciesThermo object to calculate the
80  * reference state thermodynamic properties. Then, the pressure dependence is
81  * added in within the child ThermoPhase object to complete the specification of
82  * the standard state. The VPStandardStateTP class, however, redefines the calls
83  * to the calculation of standard state properties to use VPSSMgr class calls.
84  * A listing of these classes and important pointers are supplied below.
85  *
86  * - ThermoPhase
87  * - \link Cantera::ThermoPhase::m_spthermo m_spthermo\endlink
88  * This is a pointer to a SpeciesThermo manager class that
89  * handles the reference %state Thermodynamic calculations.
90  * - VPStandardStateTP (inherits from ThermoPhase)
91  * - \link Cantera::ThermoPhase::m_spthermo m_spthermo\endlink
92  * SpeciesThermo manager handling reference %state Thermodynamic calculations.
93  * may or may not be used by the VPSSMgr class. For species
94  * which don't have a reference state class defined, a default
95  * class, called STITbyPDSS which is installed into the SpeciesThermo
96  * class, actually calculates reference state
97  * thermo by calling a PDSS object.
98  * - \link Cantera::VPStandardStateTP::m_VPSS_ptr m_VPSS_ptr\endlink
99  * This is a pointer to a VPSSMgr class which handles the
100  * standard %state thermo calculations. It may
101  * or may not use the pointer, m_spthermo, in its calculations.
102  *
103  * The following classes inherit from VPSSMgr. Each of these classes handle
104  * multiple species and by definition all of the species in a phase. It is a
105  * requirement that a VPSSMgr object handles all of the species in a phase.
106  *
107  * - VPSSMgr_IdealGas
108  * - standardState model = "IdealGas"
109  * - This model assumes that all species in the phase obey the
110  * ideal gas law for their pressure dependence. The manager
111  * uses a SpeciesThermo object to handle the calculation of the
112  * reference state.
113  * - VPSSMgr_ConstVol
114  * - standardState model = "ConstVol"
115  * - This model assumes that all species in the phase obey the
116  * constant partial molar volume pressure dependence.
117  * The manager uses a SpeciesThermo object to handle the
118  * calculation of the reference state.
119  * - VPSSMgr_Water_ConstVol
120  * - standardState model = "Water_ConstVol"
121  * - This model assumes that all species but one in the phase obey the
122  * constant partial molar volume pressure dependence.
123  * The manager uses a SpeciesThermo object to handle the
124  * calculation of the reference state for those species.
125  * Species 0 is assumed to be water, and a real equation
126  * of state is used to model the T, P behavior.
127  * - VPSSMgr_Water_HKFT
128  * - standardState model = "Water_HKFT"
129  * - This model assumes that all species but one in the phase obey the
130  * HKFT equation of state.
131  * Species 0 is assumed to be water, and a real equation
132  * of state is used to model the T, P behavior.
133  * - VPSSMgr_General
134  * - standardState model = "General"
135  * - This model is completely general. Nothing is assumed at this
136  * level. Calls consist of loops to PDSS property evaluations.
137  *
138  * The choice of which VPSSMgr object to be used is implicitly made by %Cantera
139  * by querying the XML data file for compatibility. However, each of these
140  * VPSSMgr objects may be explicitly requested in the XML file by adding in the
141  * following XML node into the thermo section of the phase XML Node. For
142  * example, the code example listed below explicitly requests that the
143  * VPSSMgr_IdealGas object be used to handle the standard state thermodynamics
144  * calculations.
145  *
146  * @code
147  * <phase id="Silane_Pyrolysis" dim="3">
148  * . . .
149  * <thermo model="VPIdealGas">
150  * <standardState model="IdealGas">
151  * <\thermo>
152  * . . .
153  * <\phase>
154  * @endcode
155  *
156  * If it turns out that the VPSSMgr_IdealGas class can not handle the standard
157  * state calculation, then %Cantera will fail during the instantiation phase
158  * printing out an informative error message.
159  *
160  * In the source code listing above, the thermo model, VPIdealGas ,was
161  * requested. The thermo model specifies the type of ThermoPhase object to use.
162  * In this case the object IdealSolnGasVPSS (with the ideal gas suboption) is
163  * used. IdealSolnGasVPSS inherits from VPStandardStateTP, so that it actually
164  * has a VPSSMgr pointer to be specified. Note, in addition to the IdealGas
165  * entry to the model parameter in standardState node, we could have also
166  * specified the "General" option. The general option will always work. An
167  * example of this usage is listed below.
168  *
169  * @code
170  * <phase id="Silane_Pyrolysis" dim="3">
171  * . . .
172  * <thermo model="VPIdealGas">
173  * <standardState model="General">
174  * <\thermo>
175  * . . .
176  * <\phase>
177  * @endcode
178  *
179  * The "General" option will cause the VPSSMgr_General VPSSMgr class to be used.
180  * In this manager, the calculations are all handled at the PDSS object level.
181  * This is completely general, but, may be significantly slower.
182  *
183  * @ingroup thermoprops
184  */

186 //! Virtual base class for the classes that manage the calculation
187 //! of standard state properties for all the species in a phase.
188 /*!
189  * This class defines the interface which all subclasses must implement.
190  *
191  * Class VPSSMgr is the base class for a family of classes that compute
192  * properties of a set of species in their standard state at a range of
193  * temperatures and pressures.
194  *
195  * If #m_useTmpRefStateStorage is set to true, then the following internal
196  * arrays, containing information about the reference arrays,
197  * are calculated and kept up to date at every call.
198  *
199  * - #m_h0_RT
200  * - #m_g0_RT
201  * - #m_s0_R
202  * - #m_cp0_R
203  *
204  * The virtual function #_updateRefStateThermo() is supplied to do this and may
205  * be reimplemented in child routines. A default implementation based on the
206  * speciesThermo class is supplied in this base class.
207  * #_updateStandardStateThermo() is called whenever a reference state property
208  * is needed.
209  *
210  * When #m_useTmpStandardStateStorage is true, then the following internal
211  * arrays, containing information on the standard state properties are
212  * calculated and kept up to date.
213  *
214  * - #m_hss_RT;
215  * - #m_cpss_R;
216  * - #m_gss_RT;
217  * - #m_sss_R;
218  * - #m_Vss
219  *
220  * The virtual function #_updateStandardStateThermo() is supplied to do this and
221  * must be reimplemented in child routines, when #m_useTmpStandardStateStorage
222  * is true. It may be optionally reimplemented in child routines if
223  * #m_useTmpStandardStateStorage is false. #_updateStandardStateThermo() is
224  * called whenever a standard state property is needed.
225  *
226  * This class is usually used for nearly incompressible phases. For those
227  * phases, it makes sense to change the equation of state independent variable
228  * from density to pressure.
229  */
