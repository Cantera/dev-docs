23 /**
24  * @defgroup spthermo Species Reference-State Thermodynamic Properties
25  *
26  * The ThermoPhase object relies on classes to calculate the thermodynamic
27  * properties of the reference state for all of the species in the phase. This
28  * group describes the types and functionality of the classes that calculate
29  * the reference state thermodynamic functions within %Cantera.
30  *
31  * To compute the thermodynamic properties of multicomponent solutions, it is
32  * necessary to know something about the thermodynamic properties of the
33  * individual species present in the solution. Exactly what sort of species
34  * properties are required depends on the thermodynamic model for the solution.
35  * For a gaseous solution (i.e., a gas mixture), the species properties
36  * required are usually ideal gas properties at the mixture temperature and at
37  * a reference pressure (almost always at 1 bar). For other types of solutions,
38  * however, it may not be possible to isolate the species in a "pure" state.
39  * For example, the thermodynamic properties of, say, Na+ and Cl- in saltwater
40  * are not easily determined from data on the properties of solid NaCl, or
41  * solid Na metal, or chlorine gas. In this case, the solvation in water is
42  * fundamental to the identity of the species, and some other reference state
43  * must be used. One common convention for liquid solutions is to use
44  * thermodynamic data for the solutes in the limit of infinite dilution within
45  * the pure solvent; another convention is to reference all properties to unit
46  * molality.
47  *
48  * In defining these standard states for species in a phase, we make the
49  * following definition. A reference state is a standard state of a species in
50  * a phase limited to one particular pressure, the reference pressure. The
51  * reference state specifies the dependence of all thermodynamic functions as a
52  * function of the temperature, in between a minimum temperature and a maximum
53  * temperature. The reference state also specifies the molar volume of the
54  * species as a function of temperature. The molar volume is a thermodynamic
55  * function. A full standard state does the same thing as a reference state,
56  * but specifies the thermodynamics functions at all pressures.
57  *
58  * Whatever the conventions used by a particular solution model, means need to
59  * be provided to compute the species properties in the reference state. Class
60  * SpeciesThermo is the base class for a family of classes that compute
61  * properties of all species in a phase in their reference states, for a range
62  * of temperatures. Note, the pressure dependence of the species thermodynamic
63  * functions is not handled by this particular species thermodynamic model.
64  * SpeciesThermo calculates the reference-state thermodynamic values of all
65  * species in a single phase during each call.
66  *
67  * The class SpeciesThermoInterpType is a pure virtual base class for
68  * calculation of thermodynamic functions for a single species in its reference
69  * state. The following classes inherit from SpeciesThermoInterpType.
70  *
71  * - NasaPoly1 in file NasaPoly1.h
72  * - This is a one zone model, consisting of a 7
73  * coefficient NASA Polynomial format.
74  * .
75  * - NasaPoly2 in file NasaPoly2.h
76  * - This is a two zone model, with each zone consisting of a 7
77  * coefficient NASA Polynomial format.
78  * .
79  * - ShomatePoly in file ShomatePoly.h
80  * - This is a one zone model, consisting of a 7
81  * coefficient Shomate Polynomial format.
82  * .
83  * - ShomatePoly2 in file ShomatePoly.h
84  * - This is a two zone model, with each zone consisting of a 7
85  * coefficient Shomate Polynomial format.
86  * .
87  * - ConstCpPoly in file ConstCpPoly.h
88  * - This is a one-zone constant heat capacity model.
89  * .
90  * - Mu0Poly in file Mu0Poly.h
91  * - This is a multi-zone model. The chemical potential is given
92  * at a set number of temperatures. Between each temperature
93  * the heat capacity is treated as a constant.
94  * .
95  * - Nasa9Poly1 in file Nasa9Poly1.h
96  * - This is a one zone model, consisting of the 9
97  * coefficient NASA Polynomial format.
98  * .
99  * - Nasa9PolyMultiTempRegion in file Nasa9PolyMultiTempRegion.h
100  * - This is a multiple zone model, consisting of the 9
101  * coefficient NASA Polynomial format in each zone.
102  * .
103  * - STITbyPDSS in file SpeciesThermoInterpType.h
104  * - This is an object that calculates reference state thermodynamic
105  * functions by relying on a pressure dependent
106  * standard state object (i.e., a PDSS object) to calculate
107  * the thermodynamic functions.
108  *
109  * The most important member function for the SpeciesThermoInterpType class is
110  * the member function SpeciesThermoInterpType::updatePropertiesTemp(). The
111  * function calculates the values of Cp, H, and S for the specific species
112  * pertaining to this class. It takes as its arguments the base pointer for the
113  * vector of Cp, H, and S values for all species in the phase. The offset for
114  * the species is known within the object.
115  *
116  * A key concept for reference states is that there is a maximum and a minimum
117  * temperature beyond which the thermodynamic formulation isn't valid. Calls
118  * for temperatures outside this range will cause the object to throw a
119  * CanteraError.
120  *
121  * @ingroup thermoprops
122  */

124 //! Pure Virtual Base class for the thermodynamic manager for an individual
125 //! species' reference state
126 /*!
127  * This differs from the SpeciesThermo virtual base class in the sense that this
128  * class is meant to handle only one species. The speciesThermo class is meant
129  * to handle the calculation of all the species (or a large subset) in a phase.
130  *
131  * One key feature is that the update routines use the same form as the update
132  * routines in the speciesThermo class. They update into a vector of cp_R, s_R,
133  * and H_R that spans all of the species in a phase. Therefore, this class must
134  * carry along a species index into that vector.
135  *
136  * These routine may be templated. A key requirement of the template is that
137  * there is a constructor with the following form:
138  *
139  * @code
140  * SpeciesThermoInterpType(int index, doublereal tlow, doublereal thigh,
