21 //! A class for banded matrices, involving matrix inversion processes.
22 //! The class is based upon the LAPACK banded storage matrix format.
23 /*!
24  * An important issue with this class is that it stores both the original data
25  * and the LU factorization of the data. This means that the banded matrix typically
26  * will take up twice the room that it is expected to take.
27  *
28  * QR factorizations of banded matrices are not included in the original LAPACK work.
29  * Add-ons are available. However, they are not included here. Instead we just use the
30  * stock LU decompositions.
31  *
32  * This class is a derived class of the base class GeneralMatrix. However, within
33  * the oneD directory, the class is used as is, without reference to the GeneralMatrix
34  * base type.
35  */
36 class BandMatrix : public GeneralMatrix

242  //! Return a pointer to the top of column j, column values are assumed to be contiguous in memory
243  /*!
244  * The LAPACK bandstructure has column values which are contiguous in memory:
245  *
246  * On entry, the matrix A in band storage, in rows KL+1 to
247  * 2*KL+KU+1; rows 1 to KL of the array need not be set.
248  * The j-th column of A is stored in the j-th column of the
249  * array AB as follows:
250  * AB(KL + KU + 1 + i - j,j) = A(i,j) for max(1, j - KU) <= i <= min(m, j + KL)
251  *
252  * This routine returns the position of AB(1,j) (fortran-1 indexing) in the above format
253  *
254  * So to address the (i,j) position, you use the following indexing:
255  *
256  * double *colP_j = matrix.ptrColumn(j);
257  * double a_i_j = colP_j[kl + ku + i - j];
258  *
259  * @param j Value of the column
260  *
261  * @return Returns a pointer to the top of the column
262  */
263  virtual doublereal* ptrColumn(size_t j);
